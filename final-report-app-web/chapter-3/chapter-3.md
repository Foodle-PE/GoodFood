## 3.2. User Stories.



| Epic / Story ID | Título | Descripción | Criterios de Aceptación | Relacionado con (Epic ID) |
| :---- | :---- | :---- | :---- | :---- |
| EP01 | Monitoreo inteligente del inventario | Funcionalidades que permiten visualizar, controlar y mantener el estado del inventario en tiempo real. |  |  |
| US01 | Monitoreo en tiempo real | Como gerente. Quiero monitorear las condiciones del inventario en tiempo real. Para evitar pérdidas por mal almacenamiento. | Escenario 01: Actualización de estado. Dado que el sistema esté conectado a sensores IoT, Cuando se detecte un cambio en temperatura, Entonces se actualiza el estado en la app instantáneamente. Escenario 02: Visualización de estado. Dado que los sensores funcionen correctamente, Cuando accedo a la app, Entonces visualizo la temperatura y humedad actuales de los alimentos. | EP01 |
| US02 | Lectura de temperatura desde sensores | Como gerente. Quiero conocer la temperatura del almacén. Para asegurar condiciones óptimas. | Escenario 01: Visualización de temperatura. Dado que el sensor está activo, Cuando abro la app, Entonces me muestra la temperatura en grados. Escenario 02: Alerta por temperatura anómala. Dado que la temperatura es inadecuada, Cuando supera los límites, Entonces se emite una alerta. | EP01 |
| US03 | Historial de temperaturas | Como administrador. Quiero ver un historial de temperatura por producto. Para detectar posibles fallas. | Escenario 01: Revisión ante producto dañado. Dado que un alimento se estropea, Cuando consulto su historial, Entonces veo si hubo cambios bruscos de temperatura. Escenario 02: Consulta por fecha. Dado que accedo a los datos, Cuando selecciono una fecha, Entonces se me muestra la evolución. | EP01 |
| US04 | Análisis de tendencias de consumo | Como administrador. Quiero ver estadísticas de consumo. Para planificar mejor mis compras. | Escenario 01: Consulta de productos más usados. Dado que reviso el módulo de análisis, Cuando selecciono un rango de fechas, Entonces veo qué productos se usan más. Escenario 02: Ajuste de pedidos. Dado que veo un producto con alta rotación, Cuando planifico compras, Entonces ajusto el pedido según el historial. | EP01 |
| US05 | Registro de condiciones de almacenamiento | Como supervisor. Quiero saber si las condiciones fueron correctas durante el día. Para evitar riesgos sanitarios. | Escenario 01: Revisión diaria de condiciones. Dado que reviso el registro diario, Cuando detecto temperaturas fuera del rango, Entonces se marca en rojo ese día. Escenario 02: Informe de cumplimiento. Dado que todo estuvo estable, Cuando cierro el día, Entonces se genera un informe de cumplimiento. | EP01 |
| EP02 | Gestión proactiva de insumos | Funciones para automatizar la gestión del stock, prevenir pérdidas y facilitar el abastecimiento. |  |  |
| US06 | Alertas por fecha de caducidad | Como encargado de almacén. Quiero recibir alertas antes de que un alimento caduque. Para evitar su desperdicio. | Escenario 01: Alerta previa a caducidad. Dado que un producto esté próximo a caducar, Cuando falten menos de 3 días, Entonces el sistema me envía una notificación. Escenario 02: Alerta de producto caducado. Dado que un alimento ya haya caducado, Cuando ingreso al inventario, Entonces se visualiza con una alerta roja. | EP02 |
| US07 | Reportes automáticos de inventario | Como administrador. Quiero recibir reportes automáticos de stock. Para optimizar mis compras semanales. | Escenario 01: Envío de reportes programados. Dado que llegue fin de semana, Cuando sea sábado a las 8 am, Entonces recibo un reporte por correo. Escenario 02: Identificación de productos a reabastecer. Dado que tengo bajo stock, Cuando reviso el reporte, Entonces identifico los productos a reabastecer. | EP02 |
| US08 | Visualización de niveles de stock | Como encargado de compras. Quiero ver el nivel actual de stock. Para saber cuándo hacer pedidos. | Escenario 01: Stock bajo visualizado. Dado que un producto está por agotarse, Cuando reviso el panel, Entonces lo veo en color naranja o rojo. Escenario 02: Stock suficiente visualizado. Dado que hay suficiente stock, Cuando veo el tablero, Entonces el producto aparece en verde. | EP02 |
| US09 | Configuración de alertas personalizadas | Como administrador. Quiero configurar las alertas según mis necesidades. Para priorizar productos críticos. | Escenario 01: Configuración de umbrales. Dado que accedo a configuración, Cuando selecciono un producto, Entonces puedo definir umbrales de alerta. Escenario 02: Notificación personalizada. Dado que configuro alertas por caducidad, Cuando se cumple el plazo, Entonces recibo la notificación según mi configuración. | EP02 |
| US10 | Registro de entrada de productos | Como encargado. Quiero registrar la llegada de productos. Para tener un control preciso del inventario. | Escenario 01: Registro de nuevo lote. Dado que llega un nuevo lote, Cuando ingreso los datos, Entonces el inventario se actualiza automáticamente. Escenario 02: Notificación de omisión. Dado que olvido registrar, Cuando reviso el inventario, Entonces me aparece una notificación de producto sin ingreso. | EP02 |
| US11 | Registro de salida de productos | Como trabajador. Quiero registrar qué productos se usan. Para mantener actualizado el stock. | Escenario 01: Registro de uso. Dado que utilizo un ingrediente, Cuando marco su salida, Entonces el inventario lo descuenta. Escenario 02: Auditoría de omisiones. Dado que un producto no se marca, Cuando hay diferencias, Entonces se registra en una auditoría. | EP02 |
| US12 | Notificaciones de stock crítico | Como responsable de compras. Quiero recibir alertas cuando un insumo esté por agotarse. Para evitar faltantes. | Escenario 01: Alerta de stock crítico. Dado que un producto alcanza el nivel mínimo, Cuando reviso la app, Entonces recibo una alerta de stock crítico. Escenario 02: Visualización de cantidades. Dado que recibo la alerta, Cuando ingreso al inventario, Entonces puedo ver exactamente qué cantidad queda. | EP02 |
| US13 | Reducción de desperdicio alimentario | Como gerente. Quiero reducir el desperdicio. Para disminuir mis costos. | Escenario 01: Alerta de caducidad próxima. Dado que se detectan productos a punto de vencer, Cuando recibo la alerta, Entonces los priorizo en la cocina. Escenario 02: Evaluación mensual de reducción. Dado que aplico el sistema, Cuando pasa un mes, Entonces los desperdicios se reducen al menos un 20%. | EP02 |
| US14 | Integración con sistema de pedidos | Como administrador. Quiero que el sistema se conecte con mi proveedor. Para generar pedidos automáticamente. | Escenario 01: Generación de pre-pedido. Dado que un insumo está por agotarse, Cuando se activa la regla de pedido, Entonces se genera un pre-pedido al proveedor. Escenario 02: Actualización tras recepción. Dado que el pedido se completa, Cuando llega el producto, Entonces se actualiza automáticamente el inventario. | EP02 |
| US15 | Control de caducidad con código QR | Como usuario. Quiero escanear productos con código QR. Para agilizar el control de fechas. | Escenario 01: Registro mediante escaneo. Dado que un producto tiene QR, Cuando lo escaneo, Entonces se registra su fecha de vencimiento. Escenario 02: Actualización automática. Dado que el sistema reconoce el código, Cuando lo almaceno, Entonces se actualiza automáticamente el inventario. | EP02 |
| US16 | Recomendaciones automáticas de uso | Como cocinero. Quiero que la app me sugiera usar productos a punto de caducar. Para evitar desperdicio. | Escenario 01: Sugerencia en pantalla principal. Dado que un producto vence pronto, Cuando inicio sesión, Entonces me aparece una recomendación para usarlo. Escenario 02: Reducción de descartes. Dado que sigo la sugerencia, Cuando uso el producto, Entonces reduzco los descartes innecesarios. | EP02 |
| EP03 | Experiencia del usuario y usabilidad | Características centradas en la facilidad de uso, accesibilidad y experiencia fluida para el personal del restaurante. |  |  |
| US17 | Interfaz intuitiva | Como trabajador de cocina. Quiero una app sencilla con iconos claros. Para usarla sin complicaciones. | Escenario 01: Navegación sencilla. Dado que ingreso a la app, Cuando navego entre secciones, Entonces entiendo fácilmente las funciones. Escenario 02: Registro rápido. Dado que necesito registrar un nuevo producto, Cuando toco el botón de agregar, Entonces el proceso no toma más de 3 pasos. | EP03 |
| US18 | Acceso multiusuario | Como dueño de restaurante. Quiero que mi equipo tenga cuentas propias. Para dividir responsabilidades. | Escenario 01: Acceso por rol. Dado que tengo diferentes roles, Cuando un cocinero accede, Entonces solo ve lo relevante a cocina. Escenario 02: Gestión de accesos. Dado que soy administrador, Cuando gestiono usuarios, Entonces puedo crear o quitar accesos. | EP04 |
| US19 | Aplicación móvil compatible | Como cocinero. Quiero acceder desde mi celular. Para consultar el inventario sin ir al almacén. | Escenario 01: Acceso desde el móvil. Dado que tengo conexión a internet, Cuando descargo la app, Entonces puedo consultar el inventario desde mi móvil. Escenario 02: Acceso sin conexión. Dado que no tengo red, Cuando accedo a la app, Entonces veo los datos más recientes en caché. | EP03 |
| US20 | Visualización por categorías de producto | Como usuario. Quiero ver los productos agrupados. Para organizar mejor el inventario. | Escenario 01: Filtro por categoría. Dado que accedo al inventario, Cuando aplico un filtro, Entonces puedo ver solo carnes, verduras, etc. Escenario 02: Búsqueda eficiente. Dado que hay muchos productos, Cuando uso el buscador, Entonces los resultados se ordenan por relevancia. | EP03 |
| US21 | Soporte multilenguaje | Como trabajador no hispanohablante. Quiero poder usar la app en inglés. Para entender mejor las funciones. | Escenario 01: Cambio de idioma. Dado que cambio el idioma, Cuando selecciono inglés, Entonces toda la interfaz se adapta. Escenario 02: Idioma por usuario. Dado que varios trabajadores usan la app, Cuando acceden con su cuenta, Entonces la app se muestra en su idioma elegido. | EP03 |
| EP04 | Configuración y administración | Funcionalidades de configuración, seguridad, trazabilidad y administración del sistema por parte del restaurante. |  |  |
| US22 | Adaptabilidad del sistema | Como dueño de restaurante. Quiero que el sistema se adapte a mis instalaciones. Para que funcione sin importar el tamaño. | Escenario 01: Instalación en local mediano. Dado que tengo un restaurante mediano, Cuando instalo los sensores, Entonces se integran sin necesidad de infraestructura extra. Escenario 02: Escalabilidad a cadena. Dado que crezco a una cadena, Cuando amplío el sistema, Entonces la app permite escalar la solución. | EP04 |
| US23 | Soporte técnico incluido | Como usuario. Quiero tener soporte técnico. Para resolver problemas con los sensores. | Escenario 01: Solicitud de soporte. Dado que el sensor no responde, Cuando solicito ayuda, Entonces recibo soporte en menos de 24h. Escenario 02: Uso del chat de ayuda. Dado que tengo una duda, Cuando uso el chat de ayuda, Entonces recibo orientación clara. | EP04 |
| US24 | Reporte de productos en mal estado | Como supervisor. Quiero reportar alimentos dañados. Para hacer seguimiento a sus causas. | Escenario 01: Registro de producto dañado. Dado que detecto un alimento podrido, Cuando lo marco como dañado, Entonces se crea un registro. Escenario 02: Análisis de recurrencias. Dado que analizo el reporte, Cuando veo recurrencias, Entonces identifico patrones y ajusto el proceso. | EP04 |
| US25 | Integración con hojas de cálculo | Como administrador. Quiero exportar los datos a Excel. Para analizarlos con mi equipo. | Escenario 01: Exportación de inventario. Dado que accedo al módulo de reportes, Cuando selecciono exportar, Entonces obtengo un archivo .xlsx. Escenario 02: Análisis con filtros. Dado que edito en Excel, Cuando realizo filtros, Entonces visualizo las tendencias de consumo. | EP04 |
| US26 | Entrenamiento inicial | Como nuevo usuario. Quiero recibir una capacitación. Para usar la plataforma correctamente. | Escenario 01: Capacitación programada. Dado que recién contrato el sistema, Cuando llega el día de capacitación, Entonces el equipo me enseña en 1 hora. Escenario 02: Aplicación práctica. Dado que termino el entrenamiento, Cuando empiezo a usar la app, Entonces entiendo todas las funciones básicas. | EP04 |
| US27 | Módulo de auditoría | Como gerente. Quiero saber quién hizo cambios. Para tener trazabilidad del inventario. | Escenario 01: Revisión de historial. Dado que se edita un producto, Cuando reviso la auditoría, Entonces veo quién, cuándo y qué modificó. Escenario 02: Corrección de errores. Dado que hay errores, Cuando reviso el historial, Entonces puedo corregirlos. | EP04 |
| US28 | Planes de suscripción escalables | Como dueño de restaurante. Quiero elegir entre diferentes planes. Para pagar solo por lo que necesito. | Escenario 01: Selección de plan básico. Dado que soy un local pequeño, Cuando contrato el servicio, Entonces selecciono el plan básico. Escenario 02: Escalado de plan. Dado que abro más sucursales, Cuando actualizo el plan, Entonces accedo a funciones premium. | EP04 |
| US29 | Soporte en español por WhatsApp | Como usuario. Quiero contactar soporte por WhatsApp en español. Para resolver dudas rápidamente. | Escenario 01: Atención por mensaje. Dado que tengo un problema, Cuando escribo al número de soporte, Entonces me responde un agente en español. Escenario 02: Solución rápida. Dado que explico el problema, Cuando recibo la respuesta, Entonces soluciono el inconveniente en minutos. | EP04 |
| US30 | Evaluación del sistema por los usuarios | Como administrador. Quiero que los usuarios califiquen el sistema. Para saber su nivel de satisfacción. | Escenario 01: Solicitud de evaluación. Dado que pasa un mes de uso, Cuando el usuario abre la app, Entonces se le pide que evalúe su experiencia. Escenario 02: Mejora basada en feedback. Dado que recibimos feedback, Cuando lo analizamos, Entonces mejoramos la plataforma según los comentarios. | EP04 |
